## 実装要件

### 1. HTML / CSS 設計

- BEM（Block Element Modifier）命名規則に基づく保守性の高いCSS設計
- クライアントがテキスト修正等を行う際、構造を壊さず管理しやすい設計とする
- PC / SP レスポンシブ対応（全ページ）

### 2. GTM 対応

- 各CTAボタン等へのカスタム属性（data属性）の付与
- 計測ポイントの設計はクライアント側で行い、「計測ポイント指示一覧」（PDF またはスプレッドシート）として提供される
- 実装者は指示一覧に従いdata属性を付与する

### 3. パフォーマンス最適化

- WebP 画像対応
- lazy load（ファーストビュー外の画像に `loading="lazy"` 付与）
- preload（LCP対象のファーストビュー画像を先読み）
- srcset によるレスポンシブ画像（必要に応じて）
- PageSpeed Insights モバイルスコア 85〜90点 **（目標値であり保証ではない）**
- デザインの構成（画像点数・ファーストビュー構成・Webフォント使用有無）によって達成難易度が変動する

### 4. 子ページ用汎用テンプレート

- 共通ヘッダー・フッター・CSS を流用した空テンプレート 1枚
- クライアントが特商法等の子ページを自作できる構成とする
